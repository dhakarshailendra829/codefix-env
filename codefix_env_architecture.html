What this project does

CodeFix-Env is a reinforcement learning environment where an AI agent (LLM) learns to automatically find and fix bugs in Python code. The agent observes buggy code and test failure output, takes editing actions, and receives reward signals based on how many tests now pass. Built on the Meta PyTorch OpenEnv framework for the OpenEnv hackathon.

The core idea: instead of manually labelling what a "good" fix looks like, we let the agent discover fixes through trial-and-error, guided by the ground-truth test suite.

Core RL loop

Standard gymnasium-style interaction loop. The agent acts, the environment executes the action in a sandboxed subprocess, reward is computed, and the agent updates its policy.

System prompt design

1
Model must output only valid JSON — no preamble, no markdown fences. Pydantic validation enforces schema.
2
Full action schema: action_type, line_number, new_content, reasoning. All types specified.
3
Numbered code lines in prompt minimise positional confusion. Test output shown verbatim.

GRPO reward signal

1
Each candidate action executed in isolated sandboxed subprocess.
2
Rewards normalised within group: relative advantage = (r − mean) / std.
3
Policy gradient update with KL penalty to prevent catastrophic forgetting.

Hallucination avoidance

LLMs acting as code-editing agents have specific failure modes. The project addresses each one structurally through reward signals and environment design.

Phantom line numbers
LLM references line 42 in a 12-line file.
Fix: numbered code in every prompt. edit_line(invalid) returns error reward −0.02. Model learns to count lines first.
Inventing test names
Model fabricates test case names that don't exist.
Fix: full test output with real names in every observation. Ground truth always visible in prompt.
Incorrect JSON schema
Model outputs wrong field names or malformed JSON.
Fix: Pydantic v2 validation on every action. Parse failure → −0.1 reward. Model converges to correct schema.
Confident wrong fixes
Model submits a fix it believes is correct but tests fail.
Fix: run_tests before submit provides ground-truth feedback. Model learns to verify before submitting.
Importing non-existent modules
Model inserts import custom_utils which crashes.
Fix: sandbox import whitelist (stdlib only). Blocked imports surface in test_output as clear error. Negative reward steers away.
Reasoning-action mismatch
Model says "edit line 3" but submits action for line 5.
Fix: reasoning field is logged only, never executed. Only action fields are trusted and executed.

Reward engineering

Two reward sources blended together. Rule-based is always active. Neural model is optional, loaded from a checkpoint file.

Rule-based reward (always active)
Per step: (tests_passed_delta / total) × (1 + α)
Solve bonus: +0.20 when all tests pass
Time decay: reward × γ^step_count (γ=0.99)
Hint penalty: −0.05 per hint used
Invalid action: −0.02
RewardMLP (optional neural model)
Input: 8 features — test ratio, step ratio, hints, diff score, prev score, action type, code length ratio, test count
Network: Linear(64) → LayerNorm → ReLU → Dropout → Linear(64) → LayerNorm → ReLU → Linear(1) → Sigmoid
Blend: (1−w) × rule + w × neural (w=0.3 default)

Sandbox security model

Every agent-generated code edit runs in a fully isolated subprocess. Protection is multi-layered.

Execution pipeline (5 layers)

1
AST parse — static check before execution. Blocks exec, eval, compile, __import__.
2
Import whitelist — only stdlib math, collections, itertools, functools etc. os/subprocess/socket blocked.
3
Restricted builtins dict — only safe Python built-ins passed to exec namespace.
4
Child process (daemon=True). Parent joins with timeout. Process killed if timeout exceeded.
5
stdout/stderr captured via io.StringIO. Result passed via multiprocessing.Queue. No state leaks.
Allowed imports
math, cmath, decimal, fractions, random, statistics, itertools, functools, operator, collections, heapq, bisect, array, copy, re, string, textwrap, difflib, json, enum, dataclasses, typing, abc, io
Blocked (SecurityError)
os, sys, subprocess, socket, requests, urllib, pathlib, shutil, importlib, ctypes, eval, exec, compile, __import__
Timeout behaviour
Default 5s per test case. Infinite loops killed. timed_out=True in ExecutionResult. −0.05 reward.
